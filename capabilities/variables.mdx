In this guide, we'll introduce you to Phonely's powerful variable system that serves as the backbone for capturing, storing, and reusing information throughout your call workflows.

The **Variables** system serves as a vital tool for personalizing conversations, powering integrations, and automating follow-ups with dynamic data capture and storage capabilities.

<Callout type="info">
Variables are the backbone of how Phonely agents capture, store, and reuse information during live calls and for post-call reporting and follow-ups.
</Callout>

## Key Features

1. **Pre-Built Variables** - Automatically available on every call with essential call data
2. **Custom Variables** - User-defined values that can be reused across workflows
3. **Auto-Gather Variables** - Capture caller input at the moment it's needed
4. **Post-Call Variables** - Comprehensive data available after call completion
5. **AI-Generated Metadata** - Automatic extraction of insights and sentiment analysis

## Pre-Built Variables

Automatically available on every call:

<Properties>
  <Property name="Phone number" type="string">The purchased Phonely number the call came in on.</Property>
  <Property name="Customer phone number" type="string">The caller's actual number.</Property>
  <Property name="Call start time" type="datetime">Exact start time of the call.</Property>
  <Property name="Call date" type="date">Date the call began.</Property>
  <Property name="Caller ID" type="string">Standard telephony caller ID data.</Property>
  <Property name="Caller city / ZIP" type="string">Geographic details inferred from the number's area code.</Property>
</Properties>

These are passed from the telephony system automatically and require no configuration.

## Custom Variables

Custom variables are user-defined values that can be reused across workflows for consistent configuration.

**Key Benefits:**
- **Consistent Configuration** - Set values once and use across multiple workflows
- **Flexible Data Types** - Support for text, numbers, emails, phone numbers, and more
- **API Integration** - Store API keys and configuration data for seamless integrations

**Example Use Case**: Store your API key once as a variable and use it in multiple API request blocks.

## Auto-Gather Variables

Auto-gather captures caller input at the moment it's needed instead of requiring a separate block.

<Tabs>
  <Tab title="Example">
    In an API request block, define caller_name as an auto-gather variable. At runtime, Phonely will prompt the caller for their name before sending the request.
  </Tab>
</Tabs>

<Properties>
  <Property name="Supported types" type="array">text, number, email, phone, date, time, etc.</Property>
  <Property name="Required" type="boolean">Variables can be marked as required.</Property>
</Properties>

## Post-Call Variables

After a call ends, all live variables remain available. Additional post-call variables are automatically generated:

**Caller & Agent Data** - Essential identification and workflow information
<Properties>
  <Property name="Phone number" type="string">Purchased Phonely number.</Property>
  <Property name="Customer phone number" type="string">Inbound caller's number.</Property>
  <Property name="Agent ID" type="string">Unique ID assigned to the Phonely agent.</Property>
  <Property name="Agent name" type="string">The agent's configured name.</Property>
  <Property name="Workflow ID" type="string">The unique ID of the flow used.</Property>
  <Property name="Workflow name" type="string">Display name of the active workflow.</Property>
</Properties>

**Transcript & Recording** - Conversation data and media access
<Properties>
  <Property name="Transcript JSON" type="object">Structured transcript data.</Property>
  <Property name="Transcript text" type="string">Plain text transcript.</Property>
  <Property name="Recording URL" type="url">Raw link to the call audio file.</Property>
  <Property name="Dashboard URL" type="url">Link to the full Phonely dashboard view with transcript, variables, and analytics.</Property>
</Properties>

**AI-Generated Metadata** - Intelligent analysis and insights
<Properties>

  <Property name="Sentiment" type="enum">Neutral, happy, frustrated, sad, etc.</Property>
  <Property name="Call topic" type="string">AI classification of main topic.</Property>
  <Property name="Call purpose" type="string">AI classification of purpose.</Property>
  <Property name="Key points" type="array">3â€“5 bullet highlights from the conversation.</Property>
</Properties>

**Extracted Entities** - Automatically captured structured data
Phonely automatically extracts structured data from conversations:

<Properties>
  <Property name="Dates" type="array">Any dates mentioned.</Property>
  <Property name="Times" type="array">Any times mentioned.</Property>
  <Property name="Emails" type="array">Any email addresses mentioned.</Property>

**Follow-Up & Action Items** - Post-call automation and task management
<Properties>
  <Property name="Unanswered questions" type="array">Questions the AI couldn't resolve.</Property>

**Call Metadata** - Timing and identification data
<Properties>
  <Property name="Call direction" type="enum">Inbound or outbound.</Property>
  <Property name="Call start" type="datetime">Timestamp.</Property>
  <Property name="Call end" type="datetime">Timestamp.</Property>
  <Property name="Call duration" type="duration">Total length.</Property>
  <Property name="Call ID" type="string">Unique identifier for the call.</Property>

**Outcome Tracking** - Success metrics and call analysis
<Properties>
  <Property name="Outcome" type="string">Tagged outcome (AI or manual). Example: "appointment booked" or "support resolved."</Property>

        <li>Transferred</li>
        <li>Silence timeout</li>
        <li>Voicemail detected (for outbound)</li>
        <li>Greeting hang-up</li>
      </ul>
    </Accordion>
  </Property>
  <Property name="Last block" type="string">The last block reached before call ended.</Property>
</Properties>

**A/B Testing Variables** - Testing framework data
<Properties>

## Best Practices

- **Use descriptive variable names** - Choose clear, meaningful names for your custom variables
- **Leverage auto-gather for dynamic input** - Capture caller information exactly when needed
- **Utilize post-call variables for analytics** - Take advantage of AI-generated insights for reporting
- **Test variable configurations** - Ensure variables work correctly before deploying workflows
- **Document variable purposes** - Keep track of what each variable is used for in your workflows

## Why Variables Matter

Variables enable you to:

- **Personalize conversations dynamically** - Use caller data to create tailored experiences
- **Power integrations** - Pass caller details into CRMs and other systems via API
- **Automate follow-ups** - Use post-call data to trigger automated follow-up actions
- **Track outcomes and performance** - Analyze sentiment, success rates, and call metrics for continuous improvement

<Warning>
Ensure all variables are correctly configured to maximize call handling efficiency and data accuracy.
</Warning>

Phonely's comprehensive variable system provides the foundation for creating dynamic, personalized, and data-driven call experiences.
